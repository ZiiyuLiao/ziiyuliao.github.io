MVC架構

何謂MVC

MVC是一種「網路架構流程」

Route(路由控制)

  • 將收到的訊息，轉發到該去的地方。
  • 根據Client端要求的網址，Route將所要求邏輯，轉發給對應的Controller進行處理。

Controller(控制器)

  • 用來處理網頁要求的「應用程式邏輯」
  • 介於Model與View之間

Model(資料模型)

  • 向資料庫取出資料並回傳給需求資料的Controller

View(視圖)

  • 用來處理網頁要求之「畫面呈現邏輯」。
  • 常見的就是CSS 、HTML、JS…等呈現畫面有關。

MVC全部流程

  • 整個網站的入口就是位於 /public/index.php ，由它去叫laravel開始動作，然後根據Client端要求的網址，Route會去轉給對應的Controller，經過處理後回傳View給Client。

專案結構

  • app - 網站內的controllers（控制器），model（模型），view（視圖）和 assets都是放在這個資料夾中

    • Console - 包含應用程式所有自定義的 Artisan 指令
    • Exception - 除錯檔案
    • http - 控制器、中介層以及表單請求。
      • controllers - 控制器檔案
      • middleware - 中間件檔案
    • providers - 套件服務(將邏輯打包成服務。常來此處更改套件的設定與使用方+ 式)

  • booststrap - 目錄放置啟動框架的 app.php 檔案。這個目錄也放置一個 cache 目錄，其中包含框架為效能最佳化所產生的檔案，像是路由和服務快取的檔案。

  • config - 設定參數(ex:連到哪個database，email發到哪個server)

  • database - 資料庫檔案(ex:製作假資料、設定啟動Server要預設產生的資料)

  • public - public 目錄放置 index.php 檔案，這是進入應用程式並自動載入設定的所有請求的入口。這個目錄也放置你的前端資源，像是圖片、JavaScript 和 CSS。

  • resources - 放置你的視圖以及原生、尚未編譯過的資源，像是 LESS、SASS 或 JavaScript。這個目錄也放置所有的語系檔案。

  • routes - 放置所有為應用程式定義的路由。web.php、api.php、console.php 和 channels.php。

  • storage - storage 目錄放置你編譯 Blade 模板、檔案形式的 Session 、檔案快取和其他由框架產生的檔案。

  • test - 放置你的自動化測試

  • vendor - 放置你的 Composer 依賴項目

  • Http>> Controllers - 控制器檔案

  • Http>> Middleware - 中間件檔案

  • Models - 資料模塊檔案

  • .env - 專案開發環境的設定檔案

基礎Routing

路由(Route)是Controller與view溝通之間的橋樑，將一個 HTTP 請求轉到指定的controller去處理，進而回傳畫面給使用者顯示。由這裡可以想像整個MVC是由Routing 來串起，Route 控制該使用哪一個Controller 與哪個 View來顯示，所以MVC的框架上通常都會有自訂的Route設定，那在laravel中則是由routes的資料夾下的web.php來做定義。

基本的路由範例

http method 支援 GET、POST、PUT、DELETE、MATCH、ANY 四種。

  • GET:讀取資料(不會變更資料)，Route::get($uri, $callback);
  • POST:新增資料，表單送出或是上傳檔案這一類的行為(新增)，Route::post($uri, $callback);
  • PUT:更新資料(更新)，Route::put($uri, $callback);
  • DELETE:刪除資料時(刪除)，Route::delete($uri, $callback);
  • MATCH:支援多個http method，Route::match([‘get’ , ‘post’],$uri, $callback);
  • ANY:支援所有http method，Route::any($uri, $callback);

重新定向

如果要重新定向到另一個URL的路由，可以使用 redirect

正規表示式規則

可以使用路由實例上的 where 方法規範路由參數的格式。

全局規範

如果希望規則套用至全站的路由參數，在RouteServiceProvider的boot中使用 pattern 方法定義規則。

正規表示式定義好之後，便會自動應用這些規則到所有使用該參數名稱的路由上

命名

藉由自行命名路由的名稱，這樣方便撰寫程式時，可以明確及簡短的呼叫路由，也不會因為程式邏輯的關係，降低程式碼的閱讀性。

Artisan

Laravel的CLI Artisan，可以利用CLI快速完成某些事情。

使用CLI的好處

  1. 快速建立基礎結構的檔案
    許多CLI可以藉由一行指令，就把基礎的單檔結構檔案建立完成，比起自己需要去建立檔案加複製，還是CLI會快一些。
  2. 功能豐富，有如裝了外掛
    CLI往往支援很多相對應的功能，可以增加在開發時期所需要的作業時間。
  3. 快速簡易啟動服務
    顧名思義，藉由幾行指令來快速重啟服務、搬移…等。

指令集

php artisan list 或者 php artisan 查看所有的指令清單。
大致能做到的事情有這幾類:

  1. 建立controller
    • php artisan make:controller CustomerController
  2. 建立migragtion
    • php artisan make:migration
  3. 建立model
    • php artisan make:model Customer
  4. 版本控制
  5. 啟動服務
    • 執行資料庫遷移檔
      • php artisan migrate
    • 退回上一次資料庫遷移動作
      • php artisan migrate:rollback
    • 產生資料庫遷移檔
      • php artisan migrate:make
    • 使用種子建立資料庫資料
      • php artisan db:seed
    • 產生路由快取
      • php artisan route:cache
    • 清除路由快取
      • php artisan route:clear
  6. 自創指令
